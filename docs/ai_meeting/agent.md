# Agent Context & Rules

## プロジェクトコンテキスト
Laravel 12 (Docker Sail環境) で構築されたポータルサイトへの機能追加。

## コーディング規約と技術スタック
- **PHP**: 8.2以上。`declare(strict_types=1);` を全ファイルに適用する。
- **Laravel**: 
  - Modelでは `$fillable` を使用すること。
  - Enumキャスト (`protected $casts`) を積極的に利用すること。
  - コントローラーは `invoke` 形式ではなく、標準的なRESTfulアクションメソッドを使用する。
- **DB**: MySQL 8.0。Migrationファイルは匿名クラス (`new class extends Migration`) を使用する。

## AIへの振る舞い指示
- 実装計画 (`plans.md`) に基づいて、ステップバイステップでコードを生成すること。
- コードを生成する際は、**ファイルパス**と**ファイル内容**を明確に提示すること。
- 既存のファイルを修正する場合は、該当箇所だけでなく「どのクラスのどのメソッドか」がわかるように提示すること。
- エラーハンドリング（Try-Catch）を忘れずに実装すること。
- **実装完了後は、必ず `exec.md` に以下の形式でログを追記すること。**
  - **日時と実行したStep**: (例: 2024-01-01 Step 1 DB設計)
  - **作成・修正したファイル一覧**:
  - **実装のポイント**: (なぜその設計にしたか等の簡易メモ)
