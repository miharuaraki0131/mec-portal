# Verification & Testing

## テスト観点 (MVP)

### 正常系
- [ ] **録音・送信**: 1分程度の音声を録音し、アップロードが成功するか？
- [ ] **DB保存**: `meetings` テーブルに `status: processing` でレコードが作成されるか？
- [ ] **AI処理**: Jobが実行され、Geminiから正しいJSONが返ってくるか？
- [ ] **結果反映**: `transcript` と `summary` がDBに保存され、`status: completed` になるか？

### 異常系
- [ ] **マイク拒否**: ブラウザでマイク許可を拒否した際にエラーメッセージが出るか？
- [ ] **ファイルサイズ**: 設定値を超える長時間録音を送信した際、適切なエラーが返るか？
- [ ] **APIエラー**: Gemini APIキーが無効な場合、Jobが失敗し `status: failed` になるか？

### セキュリティ・パフォーマンス
- [ ] `storage/app/private` の音声ファイルに直接URLでアクセスできないこと。
- [ ] 1時間の録音データ（約50MB）をアップロードしてもPHPがタイムアウトしないこと。
